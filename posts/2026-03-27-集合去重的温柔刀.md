# 集合去重的温柔刀

在日常开发中，对 `List` 进行去重是一个非常常见的操作。例如：

```java
List<String> uniqueAttachmentList = attachmentList.stream()
        .distinct()
        .collect(Collectors.toList());
```

这段代码优雅、简洁，但在某些对性能和内存敏感的场景下，并不是最优解。

本文将系统分析几种常见的去重方式，并从**实现原理、内存占用、适用场景**等角度进行对比。

------

## 一、Stream + distinct（常规写法）

### 示例代码

```java
List<String> uniqueAttachmentList = attachmentList.stream()
        .distinct()
        .collect(Collectors.toList());
```

### 原理分析

- `distinct()` 底层使用 `LinkedHashSet` 去重
- 保证元素顺序
- 内部仍然会创建额外的数据结构

### 优点

- 语法简洁，符合函数式编程风格
- 可读性强

### 缺点

- 存在 Stream 流对象创建开销
- 中间操作较多，内存利用不够极致

### 适用场景

- 普通业务开发
- 对性能要求不高的场景

------

## 二、LinkedHashSet 去重（推荐方案）

### 示例代码

```java
List<String> uniqueAttachmentList = new ArrayList<>(new LinkedHashSet<>(attachmentList));
```

### 原理分析

- `LinkedHashSet`：
  - 去重：基于 `HashSet`
  - 保序：维护双向链表
- 构造时直接完成去重

### 优点

- 一步完成去重
- 保持原始顺序
- 内存占用优于 Stream 写法
- 无额外流式对象开销

### 缺点

- 可读性略逊于 Stream（但仍然清晰）

### 适用场景

- 推荐作为默认去重方式
- 大部分业务场景首选

------

## 三、HashSet + 原地去重（最省内存）

### 示例代码

```java
Set<String> seen = new HashSet<>();
attachmentList.removeIf(e -> !seen.add(e));
```

### 原理分析

- `HashSet.add()`：
  - 返回 `true`：说明是新元素
  - 返回 `false`：说明重复元素
- `removeIf` 删除重复项
- 在原集合上直接操作

### 优点

- **不创建新 List**
- 内存占用最低
- 适合大数据量场景

### 缺点

- 会修改原始集合
- 不适用于需要保留原数据的场景
- 并发环境需谨慎

### 适用场景

- 批处理任务
- 大数据量处理（百万级以上）
- 内存敏感场景

------

## 四、HashSet 去重（不保证顺序）

### 示例代码

```java
List<String> uniqueAttachmentList = new ArrayList<>(new HashSet<>(attachmentList));
```

### 原理分析

- 使用 `HashSet` 去重
- 不维护插入顺序

### 优点

- 实现简单
- 性能较好

### 缺点

- 元素顺序会被打乱

### 适用场景

- 不关心顺序的业务

------

## 五、几种方式对比

| 方式               | 是否保序 | 是否新建集合 | 内存占用 | 推荐程度 |
| ------------------ | -------- | ------------ | -------- | -------- |
| Stream + distinct  | ✅        | ✅            | 中       | ⭐⭐⭐      |
| LinkedHashSet      | ✅        | ✅            | 较低     | ⭐⭐⭐⭐⭐    |
| removeIf + HashSet | ✅        | ❌            | 最低     | ⭐⭐⭐⭐     |
| HashSet            | ❌        | ✅            | 较低     | ⭐⭐⭐      |

------

## 六、最佳实践建议

### 1. 默认推荐

```java
List<String> uniqueAttachmentList = new ArrayList<>(new LinkedHashSet<>(attachmentList));
```

✔ 简洁
✔ 高效
✔ 保序

------

### 2. 大数据量优化

```java
Set<String> seen = new HashSet<>();
attachmentList.removeIf(e -> !seen.add(e));
```

✔ 极致节省内存
✔ 避免对象创建

------

### 3. 不关心顺序

```java
List<String> uniqueAttachmentList = new ArrayList<>(new HashSet<>(attachmentList));
```

✔ 更快
✔ 更轻量

------

## 七、总结

- `distinct()` 本质上并不神秘，它也是基于 `Set`
- 真正影响性能的，是：
  - 是否创建中间对象
  - 是否复制集合
- 在工程实践中：
  - **优先选择 LinkedHashSet**
  - **极端场景使用原地去重**

------

当数据像潮水一样涌来时，代码的每一分克制，都会换来系统的一寸从容。
去重这件小事，写得轻一点，系统就能跑得远一点。
